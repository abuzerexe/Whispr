## 4. AI Moderation — Posts

### 4.1 When It Runs

After the user taps "Post" and before saving to Firestore.

### 4.2 What to Check

Send the **topic ID**, **post title**, and **post content** to Gemini. The AI must evaluate two things:

1. **Sentiment / Content Safety Check**
   - REJECT if content contains: explicit sexual content, nudity descriptions, graphic racism, hate speech targeting a group, child exploitation references
   - ALLOW: domestic violence discussion, relationship issues, affairs, harassment experiences, trauma sharing, mental health struggles, grief, strong emotions, mild profanity used emotionally (not directed as hate)

2. **Topic Relevance Check**
   - Verify that the post content is actually related to the selected topic
   - REJECT if there is a clear mismatch (e.g., selected "AI & Technology" but posted explicit sexual content; selected "Personal Story" but posted spam/advertisements)
   - Allow reasonable overlap — a post about a relationship trauma can fit under both "Past Trauma" and "Relationships"

### 4.3 Gemini Prompt (Exact Template)

```
You are a content moderation AI for an anonymous mental health and support community app.

Evaluate the following post submission:

SELECTED TOPIC: {topicLabel}
POST TITLE: {postTitle}
POST CONTENT: {postContent}

Perform TWO checks and respond ONLY in the following JSON format (no extra text, no markdown):

{
  "contentSafe": true or false,
  "topicRelevant": true or false,
  "violations": ["list of specific violations if any, empty array if none"],
  "reason": "short explanation in 1-2 sentences"
}

CONTENT SAFETY RULES:
- Mark contentSafe as FALSE if the post contains: explicit sexual descriptions, nudity, graphic racial slurs, child exploitation, or calls for violence against a specific person or group.
- Mark contentSafe as TRUE if the post discusses: domestic violence (as a victim or witness), relationship problems, affairs, harassment experiences, trauma, grief, mental health struggles, addiction, or emotional distress — even if the language is strong or emotional.

TOPIC RELEVANCE RULES:
- Mark topicRelevant as FALSE only if there is an OBVIOUS mismatch — the post has nothing to do with the selected topic.
- Mark topicRelevant as TRUE if there is any reasonable connection, even indirect.
- Spam, advertisements, and gibberish should be marked topicRelevant as FALSE.
```

### 4.4 Response Handling

```dart
// Parse Gemini response
if (!moderationResult.contentSafe) {
  // Show error to user — do NOT post
  showViolationDialog(violations: moderationResult.violations, reason: moderationResult.reason);
  return;
}

if (!moderationResult.topicRelevant) {
  // Show error to user — do NOT post
  showTopicMismatchDialog(reason: moderationResult.reason);
  return;
}

// Both checks passed — proceed to save post in Firestore
```

### 4.5 User-Facing Error Messages

## 5. AI Moderation — Comments

### 5.1 When It Runs

After the user taps "Comment" / "Submit" and before saving the comment to Firestore.

### 5.2 What to Check

Only **Content Safety** — topic relevance check is NOT needed for comments (comments are assumed to be on-topic since they're replies to an existing post).

### 5.3 Gemini Prompt (Exact Template)

```
You are a content moderation AI for an anonymous mental health and support community app.

Evaluate the following comment:

COMMENT: {commentContent}

Check content safety and respond ONLY in the following JSON format (no extra text, no markdown):

{
  "contentSafe": true or false,
  "violations": ["list of specific violations if any, empty array if none"],
  "reason": "short explanation in 1-2 sentences"
}

CONTENT SAFETY RULES:
- Mark contentSafe as FALSE if the comment contains: explicit sexual descriptions, nudity, graphic racial slurs, child exploitation, calls for violence against a specific person or group, or targeted personal harassment/doxxing.
- Mark contentSafe as TRUE if the comment expresses: empathy, support, personal experiences, emotional reactions, disagreements (even heated), mental health discussions, relationship advice, strong opinions, or mild profanity used emotionally.
```

### 5.4 Response Handling

```dart
if (!moderationResult.contentSafe) {
  showCommentViolationSnackbar(reason: moderationResult.reason);
  return; // Do not save comment
}
// Proceed to save comment in Firestore
```

### 5.5 User-Facing Error Message

Show a **SnackBar** (less intrusive than a dialog for comments):

```
"Comment not posted: [reason from AI]. Please revise and try again."
```

## 6. Firestore Schema Changes

### 6.1 Posts Collection — Updated Document Structure

```
posts/{postId}
├── id            : String        (auto-generated)
├── authorId      : String        (existing)
├── anonymousName : String        (existing)
├── title         : String        (existing)
├── content       : String        (existing)
├── createdAt     : Timestamp     (existing)
├── topicId       : String        ← NEW (e.g., "domestic_violence")
├── topicLabel    : String        ← NEW (e.g., "Domestic Violence") [denormalized for display]
└── topicEmoji    : String        ← NEW (e.g., "🏠") [denormalized for display]
```

Comments do not need a topic field. They inherit topic context from their parent post.

| Collection | Field 1       | Field 2    | Order          |
|------------|---------------|------------|----------------|
| `posts`    | `topicId` ASC | `createdAt` DESC | — |

### 7.4 Service Class

Create `lib/services/gemini_moderation_service.dart`:

```dart
import 'dart:convert';
import 'package:http/http.dart' as http;

class ModerationResult {
  final bool contentSafe;
  final bool topicRelevant; // only used for posts
  final List<String> violations;
  final String reason;

  ModerationResult({
    required this.contentSafe,
    this.topicRelevant = true,
    required this.violations,
    required this.reason,
  });

  factory ModerationResult.fromJson(Map<String, dynamic> json) {
    return ModerationResult(
      contentSafe: json['contentSafe'] ?? true,
      topicRelevant: json['topicRelevant'] ?? true,
      violations: List<String>.from(json['violations'] ?? []),
      reason: json['reason'] ?? '',
    );
  }
}

class GeminiModerationService {
  static const String _apiKey = 'AIzaSyC3sF8XV0-Mi48MedGzREMjPJaIEjfOUyg';
  static const String _model = 'gemini-1.5-flash';
  static const String _baseUrl =
      'https://generativelanguage.googleapis.com/v1beta/models/$_model:generateContent';

  Future<ModerationResult> moderatePost({
    required String topicLabel,
    required String postTitle,
    required String postContent,
  }) async {
    final prompt = _buildPostPrompt(topicLabel, postTitle, postContent);
    return await _callGemini(prompt);
  }

  Future<ModerationResult> moderateComment({
    required String commentContent,
  }) async {
    final prompt = _buildCommentPrompt(commentContent);
    return await _callGemini(prompt);
  }

  String _buildPostPrompt(String topicLabel, String title, String content) {
    // Use the exact prompt template from Section 4.3
    return '''You are a content moderation AI for an anonymous mental health and support community app.

Evaluate the following post submission:

SELECTED TOPIC: $topicLabel
POST TITLE: $title
POST CONTENT: $content

Perform TWO checks and respond ONLY in the following JSON format (no extra text, no markdown):

{
  "contentSafe": true or false,
  "topicRelevant": true or false,
  "violations": ["list of specific violations if any, empty array if none"],
  "reason": "short explanation in 1-2 sentences"
}

CONTENT SAFETY RULES:
- Mark contentSafe as FALSE if the post contains: explicit sexual descriptions, nudity, graphic racial slurs, child exploitation, or calls for violence against a specific person or group.
- Mark contentSafe as TRUE if the post discusses: domestic violence (as a victim or witness), relationship problems, affairs, harassment experiences, trauma, grief, mental health struggles, addiction, or emotional distress — even if the language is strong or emotional.

TOPIC RELEVANCE RULES:
- Mark topicRelevant as FALSE only if there is an OBVIOUS mismatch — the post has nothing to do with the selected topic.
- Mark topicRelevant as TRUE if there is any reasonable connection, even indirect.
- Spam, advertisements, and gibberish should be marked topicRelevant as FALSE.''';
  }

  String _buildCommentPrompt(String comment) {
    // Use the exact prompt template from Section 5.3
    return '''You are a content moderation AI for an anonymous mental health and support community app.

Evaluate the following comment:

COMMENT: $comment

Check content safety and respond ONLY in the following JSON format (no extra text, no markdown):

{
  "contentSafe": true or false,
  "violations": ["list of specific violations if any, empty array if none"],
  "reason": "short explanation in 1-2 sentences"
}

CONTENT SAFETY RULES:
- Mark contentSafe as FALSE if the comment contains: explicit sexual descriptions, nudity, graphic racial slurs, child exploitation, calls for violence against a specific person or group, or targeted personal harassment/doxxing.
- Mark contentSafe as TRUE if the comment expresses: empathy, support, personal experiences, emotional reactions, disagreements (even heated), mental health discussions, relationship advice, strong opinions, or mild profanity used emotionally.''';
  }

  Future<ModerationResult> _callGemini(String prompt) async {
    try {
      final response = await http.post(
        Uri.parse('$_baseUrl?key=$_apiKey'),
        headers: {'Content-Type': 'application/json'},
        body: jsonEncode({
          'contents': [
            {
              'parts': [
                {'text': prompt}
              ]
            }
          ],
          'generationConfig': {
            'temperature': 0.1,
            'maxOutputTokens': 256,
          }
        }),
      ).timeout(const Duration(seconds: 15));

      if (response.statusCode == 200) {
        final data = jsonDecode(response.body);
        final text = data['candidates'][0]['content']['parts'][0]['text'] as String;
        // Strip any accidental markdown fences
        final cleaned = text.replaceAll(RegExp(r'```json|```'), '').trim();
        final json = jsonDecode(cleaned) as Map<String, dynamic>;
        return ModerationResult.fromJson(json);
      } else {
        // On API error, allow the post (fail open) to avoid blocking users
        return ModerationResult(contentSafe: true, topicRelevant: true, violations: [], reason: '');
      }
    } catch (e) {
      // On network/timeout error, allow the post (fail open)
      return ModerationResult(contentSafe: true, topicRelevant: true, violations: [], reason: '');
    }
  }
}
```

### 7.5 Fail-Open Policy

If the Gemini API is unreachable, times out, or returns a malformed response, **allow the post/comment through** (`contentSafe: true`). This prevents a third-party service outage from completely blocking users from posting. Log the error locally for debugging.

## 9. Error Handling & Edge Cases

| Scenario | Behavior |
|---|---|
| Gemini API timeout (>15 sec) | Fail open — allow post |
| Gemini returns invalid JSON | Fail open — allow post, log error |
| Gemini API key invalid/quota exceeded | Fail open — allow post |
| User submits empty post/comment | Existing client-side validation catches this before AI call |
| User submits very long post (>5000 chars) | Truncate content to first 3000 chars before sending to Gemini (to stay within token limits) |
| Topic not selected | "Post" button remains disabled — no AI call made |
| User taps back during AI review | Cancel the in-flight HTTP request and return to form |

## 10. Implementation Order

1. **Create `topics_constants.dart`** and `Topic` model — no side effects

6. **Add Topic Filter Bar** to home feed — update Firestore query logic
7. **Update Comments** — add AI moderation before comment save
8. **Create Firestore composite index** — do this in Firebase Console before testing filtered queries
9. **End-to-end testing** — test happy path and violation cases for both posts and comments
